# UserSettings 同步功能测试指南

## 测试目标

验证 UserSettings 的云端同步功能是否正常工作。

## 手动测试步骤

### 准备工作

1. 确保后端服务器正常运行
2. 准备两个测试账号：
   - 账号A：test_a@example.com / password123
   - 账号B：test_b@example.com / password123

---

### 测试场景 1：首次登录创建默认设置

**步骤：**
1. 使用账号A登录
2. 观察设置页面

**预期结果：**
- ✅ 页面切换动画：随机
- ✅ 对话框动画：随机
- ✅ 签到提醒：开启（20:00）
- ✅ 签到震动：开启
- ✅ 签到粒子特效：关闭

**验证方式：**
- 检查后端数据库，确认 `user_settings` 表中有账号A的记录

### 测试场景 2：修改设置自动上传

**步骤：**
1. 账号A已登录
2. 修改页面切换动画为"淡入淡出"
3. 修改对话框动画为"缩放"

**预期结果：**
- ✅ 本地设置立即更新

### 测试场景 3：登出后设置保留

**步骤：**
1. 账号A已登录，有自定义设置
2. 点击"退出登录"

**预期结果：**
- ✅ 本地设置保留（页面切换动画、对话框动画等不变）
- ✅ 继续可以修改设置（只保存到本地，不上传到云端）
- ✅ 重新登录后，云端设置会覆盖本地设置

### 测试场景 4：切换账号下载新设置

**步骤：**
1. 账号A登出
2. 使用账号B登录
3. 修改账号B的设置：
   - 页面切换动画：淡入淡出
   - 对话框动画：旋转
   - 签到提醒时间：22:00
4. 登出账号B
5. 重新登录账号A

**预期结果：**
- ✅ 登录账号B时，看到账号B的设置（或默认设置）
- ✅ 重新登录账号A时，恢复账号A之前的设置

**关键验证点：**
- 每个账号的设置是独立的
- 切换账号时，本地设置会被新账号的设置覆盖

### 测试场景 5：多设备同步

**步骤：**
1. 设备1：账号A登录，修改页面切换动画为"淡入淡出"
2. 设备2：账号A登录

**预期结果：**
- ✅ 设备2自动显示"淡入淡出"动画

**步骤（继续）：**
3. 设备2：修改对话框动画为"缩放"
4. 设备1：退出重新登录

**预期结果：**
- ✅ 设备1自动显示"缩放"动画

### 测试场景 6：冲突解决（时间戳优先 - Last Write Wins）

**场景 6.1：本地更新，上传到云端**

**步骤：**
1. 账号A登录，修改页面切换动画为"淡入淡出"
2. 等待上传成功
3. 断网（模拟离线）
4. 修改页面切换动画为"滑动"（本地保存，无法上传）
5. 恢复网络，刷新页面（触发同步）

**预期结果：**
- ✅ 页面显示"滑动"动画（本地 updatedAt 更新，上传到云端）
- ✅ 控制台日志：
  ```
  [SyncService] 本地和云端设置都存在，比较更新时间
  [SyncService] 本地设置更新，上传到云端
  ```

**场景 6.2：云端更新，覆盖本地**

**步骤：**
1. 标签页1：账号A登录，修改页面切换动画为"淡入淡出"
2. 标签页2：账号A登录（或刷新页面）
3. 标签页2：修改页面切换动画为"滑动"
4. 标签页1：刷新页面（触发同步）

**预期结果：**
- ✅ 标签页1显示"滑动"动画（云端 updatedAt 更新，覆盖本地）
- ✅ 控制台日志：
  ```
  [SyncService] 本地和云端设置都存在，比较更新时间
  [SyncService] 云端设置更新，保存到本地
  ```

## 常见问题排查

### 问题1：设置没有上传到云端

**检查：**
1. 用户是否已登录？
2. 网络是否正常？
3. 后端 API `/users/settings` 是否正常？

**调试：**
```dart
// 在 UserSettingsProvider._uploadToCloud() 中添加日志
print('上传设置到云端：${settings.toJson()}');
```

### 问题2：切换账号后设置没有更新

**检查：**
1. SyncService 是否在登录时调用了 `syncAllData()`？
2. UserSettingsProvider 是否监听了 `authProvider` 的变化？

**调试：**
```dart
// 在 UserSettingsProvider._listenToAuthChanges() 中添加日志
print('用户状态变化：$user');
```

### 问题3：多设备不同步

**检查：**
1. 两个设备是否登录了同一个账号？
2. 设置的 `updatedAt` 时间戳是否正确？
3. 冲突解决逻辑是否正常？

**调试：**
```dart
// 在 SyncService._syncUserSettings() 中添加日志
print('本地设置：${localSettings?.updatedAt}');
print('云端设置：${remoteSettings?.updatedAt}');
```

## 总结

UserSettings 同步功能已完整实现，包括：

✅ 登录时自动同步
✅ 修改时自动上传
✅ 多账号独立设置
✅ 冲突解决（Last Write Wins - 时间戳优先）
✅ 离线支持（保留离线修改）
✅ 智能冲突解决（本地更新时上传，云端更新时下载）

### 同步策略（优化后）

**全量同步 + 智能冲突解决**：
1. 云端不存在 → 上传本地设置（或创建默认设置）
2. 本地不存在 → 使用云端设置
3. 两边都存在 → 比较 `updatedAt`，使用最新的版本

**优点**：
- 保留离线修改（本地更新时上传）
- 逻辑简单（无需字段级别追踪）
- 性能优秀（全量同步，数据量小 < 1KB）
- 冲突解决合理（最后更新时间优先）

建议先进行手动测试，确认基本功能正常后，再考虑编写自动化测试。
