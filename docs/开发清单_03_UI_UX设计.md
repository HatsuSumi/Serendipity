# Serendipity 开发清单 - 03 UI/UX 设计

## 📋 UI/UX 设计检查清单

### 1. 设计理念
- [ ] 避免通用 AI 美学
- [ ] 打造独特的情感化设计
- [ ] 不使用 Inter、Roboto、Arial 等通用字体
- [ ] 避免紫色渐变等俗套配色

---

### 2. 视觉风格

#### 2.1 色彩
- [ ] 温暖、柔和、有情感张力
- [ ] 整体界面：使用用户选择的主题
- [ ] 状态卡片：根据记录状态动态应用情感色调
- [ ] 强调色：用于按钮、链接、重要提示（会员可自定义）

#### 2.2 字体
- [ ] 英文字体选择（避免通用字体）
  - [ ] Playfair Display
  - [ ] Crimson Text
  - [ ] Libre Baskerville
- [ ] 中文字体选择
  - [ ] 思源宋体
  - [ ] 霞鹜文楷
  - [ ] 站酷高端黑

#### 2.3 动画
- [ ] 流畅、有意义的过渡动画
- [ ] 状态升级动画
- [ ] 页面切换动画
- [ ] 微交互动画
- [ ] 成就解锁动画（粒子效果）

---

### 3. 情感化设计

#### 3.1 不同状态的视觉语言
- [ ] 错过🌫️：朦胧、柔和、灰蓝色调
- [ ] 再遇🌟：明亮、惊喜、金色点缀
- [ ] 邂逅💫：温暖、激动、粉橙色调
- [ ] 重逢💝：圆满、幸福、玫瑰金色调
- [ ] 别离🥀：淡然、接受、玫瑰灰色调
- [ ] 失联🍂：平静、释怀、秋叶色调

#### 3.2 状态色调应用规则
- [ ] 状态色调是情感氛围，不是整体主题
- [ ] 例如：深色主题 + 错过状态 = 深色界面 + 灰蓝色调的卡片
- [ ] 例如：温暖主题 + 邂逅状态 = 米黄色界面 + 粉橙色调的卡片

---

### 4. 交互原则

#### 4.1 快速记录
- [ ] 4步内完成记录（时间 → 地点 → 状态 → 保存）
- [ ] 除了时间、地点、状态，其他字段全部可选
- [ ] 无压力的输入体验

#### 4.2 有温度
- [ ] 文案温暖、鼓励、不说教
- [ ] 例如："还没有记录" 而不是 "暂无数据"
- [ ] 例如："点击下方按钮记录第一次错过" 而不是 "点击添加"

#### 4.3 有仪式感
- [ ] 重要时刻配合动画和音效
- [ ] 状态升级动画
- [ ] 成就解锁动画
- [ ] 匹配成功动画
