# 省市区三级联动选择器实现文档

## 📋 概述

实现了一个优雅的省市区三级联动选择器，支持搜索功能，完全遵循项目的12个代码质量原则。

## 🏗️ 架构设计

### 1. 数据模型层 (`models/region_data.dart`)

**职责**：
- 定义省市区数据结构
- 提供 JSON 序列化/反序列化
- 定义选中地区的状态模型

**遵循原则**：
- ✅ 单一职责：只负责数据结构定义
- ✅ 不可变性：使用 `const` 构造函数
- ✅ 正确实现 `==` 和 `hashCode`

### 2. 服务层 (`core/services/region_service.dart`)

**职责**：
- 加载地区数据（从 JSON 文件）
- 提供地区数据查询接口
- 提供搜索功能

**遵循原则**：
- ✅ 单一职责：只负责地区数据管理
- ✅ Fail Fast：数据加载失败立即抛出异常
- ✅ 不包含 UI 逻辑
- ✅ 参数验证：未加载数据时抛出 `StateError`

### 3. 状态管理层 (`core/providers/region_provider.dart`)

**职责**：
- 管理地区数据加载状态
- 提供响应式的数据访问接口
- 处理异步加载逻辑

**遵循原则**：
- ✅ 单一数据源：所有地区数据统一从 `RegionService` 获取
- ✅ 单向数据流：UI → Provider → Service
- ✅ 状态隔离：加载状态、省份列表、城市列表、区县列表分别管理
- ✅ 使用 `Provider.family` 实现参数化查询

### 4. UI 层 (`features/community/widgets/region_picker_dialog.dart`)

**职责**：
- 展示省市区三级联动选择器
- 提供搜索功能
- 处理用户交互

**遵循原则**：
- ✅ 不包含业务逻辑
- ✅ 不直接访问数据源
- ✅ `build()` 方法纯函数（无副作用）
- ✅ 使用 `ref.watch` 响应状态变化
- ✅ Widget 拆分：标题栏、搜索框、内容区、底部按钮分别提取

## 🎯 核心特性

### 1. 三级联动
- 选择省份 → 显示城市列表
- 选择城市 → 显示区县列表
- 支持清除选择

### 2. 搜索功能
- 实时搜索省/市/区
- 搜索结果高亮显示完整地址
- 点击搜索结果自动填充

### 3. 用户体验
- 加载状态提示
- 错误处理和重试
- 当前选择实时显示
- 清除按钮快速重置

## 🔧 使用方法

### 1. 在筛选对话框中使用

```dart
// 显示地区选择器
final result = await RegionPickerDialog.show(
  context,
  initialSelection: _selectedRegion,
);

if (result != null) {
  setState(() {
    _selectedRegion = result;
  });
}
```

### 2. 获取选中的地区信息

```dart
// 获取完整地址
String fullAddress = _selectedRegion.fullAddress;  // "广东省广州市天河区"

// 获取城市名（用于筛选）
String? cityName = _selectedRegion.city;  // "广州市"

// 检查是否完整选择
bool isComplete = _selectedRegion.isComplete;  // true/false
```
