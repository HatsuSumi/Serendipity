## 🔴 问题清单

### 问题1：_mapToInternalType 方法存在逻辑问题（中优先级 ⚡）

**违反维度**：4️⃣ Fail Fast 原则

**问题描述**：
在 `_mapToInternalType` 方法中，`none` 和 `random` 两个case返回 `_getRandomAnimationType()`，但这两个值不应该传入此方法。

**位置**：第78-82行

**当前代码**：
```dart
static _InternalAnimationType _mapToInternalType(DialogAnimationType type) {
  switch (type) {
    case DialogAnimationType.none:
    case DialogAnimationType.random:
      return _getRandomAnimationType(); // 不应该到这里
    case DialogAnimationType.fade:
      return _InternalAnimationType.fade;
    // ...
  }
}
```

**问题分析**：
1. `_selectAnimation` 方法已经处理了 `none` 和 `random`
2. 如果传入 `none` 或 `random`，说明逻辑有问题
3. 注释写着"不应该到这里"，但没有抛出异常

**修复建议**：
```dart
static _InternalAnimationType _mapToInternalType(DialogAnimationType type) {
  switch (type) {
    case DialogAnimationType.none:
    case DialogAnimationType.random:
      // Fail Fast: 这两个值不应该传入此方法
      throw AssertionError('none and random should be handled in _selectAnimation');
    case DialogAnimationType.fade:
      return _InternalAnimationType.fade;
    // ...
  }
}
```

**影响**：
- 当前代码隐藏了潜在的逻辑错误
- 违反 Fail Fast 原则
