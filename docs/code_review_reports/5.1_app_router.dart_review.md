## 🔧 修复建议

### 建议1：统一使用 GoRouter（高优先级 🔥）

**问题**：大量页面导航绕过路由系统

**建议方案**：

#### 方案A：完善 GoRouter 配置（推荐）

**优点**：
- 统一路由管理
- 支持深度链接
- 支持路由守卫
- 更好的可维护性

**缺点**：
- 需要重构现有导航代码
- GoRouter 不支持自定义页面切换动画（需要使用 CustomTransitionPage）

**工作量**：中等

---

#### 方案B：移除 GoRouter，统一使用 Navigator

**优点**：
- 与现有代码一致
- 支持自定义动画（PageTransitionBuilder）
- 无需重构

**缺点**：
- 失去 GoRouter 的优势（深度链接、路由守卫等）
- 路由管理分散

**工作量**：小

---

### 建议2：集成 PageTransitionBuilder（中优先级 ⚡）

**问题**：路由动画硬编码，无法使用用户设置

**建议**：
如果保留 GoRouter，需要在 `CustomTransitionPage` 中集成 `PageTransitionBuilder`：

```dart
pageBuilder: (context, state) {
  final record = state.extra as EncounterRecord?;
  if (record == null) {
    // 错误处理...
  }
  
  return CustomTransitionPage(
    key: state.pageKey,
    child: RecordDetailPage(record: record),
    transitionsBuilder: (context, animation, secondaryAnimation, child) {
      // 读取用户设置的动画类型
      final container = ProviderScope.containerOf(context);
      final transitionType = container.read(pageTransitionProvider);
      
      // 使用 PageTransitionBuilder
      return PageTransitionBuilder.buildTransition(
        transitionType,
        context,
        animation,
        secondaryAnimation,
        child,
      );
    },
  );
}
```
