## 四、潜在问题与建议

### 问题 1: 缺少依赖注入
**严重程度**: 中  
**描述**: 使用单例模式，难以进行单元测试。  
**当前代码**:
```dart
static final StorageService _instance = StorageService._internal();
factory StorageService() => _instance;
```
**建议**:
```dart
// 方案1: 使用 Provider 管理实例
final storageServiceProvider = Provider<StorageService>((ref) {
  final service = StorageService._internal();
  // 在测试中可以 override
  return service;
});

// 方案2: 引入接口抽象
abstract class IStorageService {
  Future<void> saveRecord(EncounterRecord record);
  // ...
}

class StorageService implements IStorageService {
  // ...
}
```
**评估**: 当前项目规模较小，单例模式可接受。如果需要单元测试，应该引入依赖注入。

---

### 问题 2: 故事线序列化方式不一致
**严重程度**: 低  
**描述**: 故事线使用 toJson/fromJson，而记录直接存储对象。  
**当前代码**:
```dart
// 记录：直接存储对象
await _recordsBoxOrThrow.put(record.id, record);

// 故事线：存储 JSON
await _storyLinesBoxOrThrow.put(storyLine.id, storyLine.toJson());
```
**建议**: 统一序列化方式，要么都用对象，要么都用 JSON。  
**评估**: 
- 记录使用 Hive Adapter，性能更好
- 故事线使用 JSON，更灵活
- 不一致但都可行，建议统一为 Hive Adapter

---

### 问题 3: 缺少批量操作
**严重程度**: 低  
**描述**: 没有批量保存、批量删除等方法。  
**建议**:
```dart
Future<void> saveRecords(List<EncounterRecord> records) async {
  final map = {for (var r in records) r.id: r};
  await _recordsBoxOrThrow.putAll(map);
}

Future<void> deleteRecords(List<String> ids) async {
  await _recordsBoxOrThrow.deleteAll(ids);
}
```
**评估**: 当前无批量操作需求，符合 YAGNI。如果未来需要，可以添加。

---

### 问题 4: close() 方法未被调用
**严重程度**: 低  
**描述**: main.dart 中初始化了 StorageService，但应用关闭时未调用 close()。  
**建议**:
```dart
// 在 MyApp 中添加生命周期管理
class MyApp extends ConsumerStatefulWidget {
  @override
  void dispose() {
    StorageService().close();
    super.dispose();
  }
}
```
**评估**: Flutter 应用关闭时系统会自动清理资源，不调用 close() 影响不大。但最佳实践是显式关闭。
