## 🔴 问题清单

### 问题1：存在死方法（低优先级 💡）

**违反维度**：7️⃣ DRY/KISS/YAGNI、8️⃣ 代码健康检查

**问题描述**：
类中定义了两个公开的getter方法，但从未被使用：

```dart
/// 快捷访问：默认浅色主题
static ThemeData get lightTheme => _lightTheme;

/// 快捷访问：默认深色主题
static ThemeData get darkTheme => _darkTheme;
```

**使用情况搜索结果**：
- `AppTheme.lightTheme` - ❌ 未找到任何调用
- `AppTheme.darkTheme` - ❌ 未找到任何调用

**实际使用方式**：
在`main.dart`中，使用的是`AppTheme.getTheme()`方法：
```dart
final lightTheme = AppTheme.getTheme(themeOption, Brightness.light);
final darkTheme = AppTheme.getTheme(themeOption, Brightness.dark);
```

**问题影响**：
- 违反YAGNI原则（You Aren't Gonna Need It）
- 增加代码维护成本
- 可能误导其他开发者

**修复建议**：
删除这两个未使用的getter：

```dart
// 删除以下代码
/// 快捷访问：默认浅色主题
static ThemeData get lightTheme => _lightTheme;

/// 快捷访问：默认深色主题
static ThemeData get darkTheme => _darkTheme;
```

### 问题2：性能可优化（低优先级 💡）

**违反维度**：9️⃣ 性能检查

**问题描述**：
每个主题getter（如`_lightTheme`、`_darkTheme`等）都会在每次调用时创建新的`ThemeData`对象。虽然Flutter会缓存ThemeData，但仍然存在不必要的对象创建。

**当前代码**：
```dart
static ThemeData get _lightTheme {
  return ThemeData(  // 每次调用都创建新对象
    useMaterial3: true,
    colorScheme: ColorScheme.fromSeed(
      seedColor: Colors.pink,
      brightness: Brightness.light,
    ),
    // ...
  );
}
```

**问题影响**：
- 每次调用`getTheme()`都会创建新的ThemeData对象
- 虽然影响不大，但不是最佳实践

**优化建议**：
使用`late final`缓存ThemeData对象：

```dart
static late final ThemeData _lightTheme = ThemeData(
  useMaterial3: true,
  colorScheme: ColorScheme.fromSeed(
    seedColor: Colors.pink,
    brightness: Brightness.light,
  ),
  // ...
);
```

**注意**：
- 这是可选优化，不是必须修复
- 当前性能影响很小
- 只有在性能分析显示瓶颈时才需要优化

## 💡 优化建议

### 建议1：删除死方法（推荐）

删除未使用的`lightTheme`和`darkTheme` getter。

**理由**：
- 违反YAGNI原则
- 增加维护成本
- 可能误导开发者

### 建议2：统一主题配置（可选）

为所有主题添加`floatingActionButtonTheme`配置：

```dart
floatingActionButtonTheme: const FloatingActionButtonThemeData(
  elevation: 4,
),
```

**理由**：
- 保持一致性
- 避免不同主题下FAB样式不一致

### 建议3：使用late final缓存（可选）

如果性能分析显示瓶颈，可以使用`late final`缓存ThemeData对象。

**当前不需要**：
- 性能影响很小
- Flutter已有内部缓存
- 过早优化
