# 📱 手机号验证配置说明

## 🎯 概述

本应用支持两种手机号验证模式：
1. **测试模式**：使用固定的测试手机号和验证码，无需真实短信
2. **生产模式**：使用自建服务器的短信验证服务，发送真实短信

---

## 🔧 配置方式

### 1. 修改配置文件

打开 `lib/core/config/app_config.dart`，修改 `enableTestPhoneNumber` 的值：

```dart
class AppConfig {
  /// 是否启用测试手机号模式
  /// 
  /// - true：使用测试手机号，固定验证码 123456（开发阶段）
  /// - false：使用自建服务器的短信验证服务（生产环境）
  static const bool enableTestPhoneNumber = true;  // ← 修改这里
}
```

### 2. 切换到真实手机号测试

**步骤 1**：将 `enableTestPhoneNumber` 改为 `false`

```dart
static const bool enableTestPhoneNumber = false;  // ✅ 禁用测试模式
```

**步骤 2**：热重载应用

**步骤 3**：使用真实手机号注册/登录

---

## 📋 两种模式对比

| 特性 | 测试模式 | 生产模式 |
|------|---------|---------|
| 配置 | `enableTestPhoneNumber = true` | `enableTestPhoneNumber = false` |
| 手机号 | `+8613800138000` | 任意真实手机号 |
| 验证码 | 固定 `123456` | 真实短信验证码 |
| 短信费用 | 免费 | 收费（短信服务商计费） |
| 适用场景 | 开发、测试 | 生产环境 |

## 🧪 测试模式详情

### 测试手机号列表

以下手机号会被识别为测试手机号（支持多种格式）：

```
+8613800138000
+86 138 0013 8000
+86-138-0013-8000
```

### 固定验证码

```
123456
```

### 工作原理

1. 用户输入测试手机号
2. 点击"发送验证码"
3. 应用检测到是测试手机号，跳过服务器验证
4. 返回一个模拟的 `verificationId`
5. 用户输入固定验证码 `123456`
6. 应用验证通过，创建模拟用户

## 🚀 生产模式详情

### 前置条件

在使用真实手机号验证之前，需要完成以下配置：

#### 1. 服务器端配置

确保自建服务器已配置短信服务：
- 配置短信服务商（如阿里云短信、腾讯云短信等）
- 配置短信模板
- 配置 API 密钥

#### 2. 客户端配置

确保 `lib/core/config/server_config.dart` 中配置了正确的服务器地址：

```dart
class ServerConfig {
  static const String baseUrl = 'https://your-server.com';
  static const String authVerificationCode = '/api/auth/verification-code';
  // ...
}
```

### 工作原理

1. 用户输入真实手机号
2. 点击"发送验证码"
3. 客户端请求自建服务器
4. 服务器调用短信服务商发送验证码
5. 用户收到短信，输入验证码
6. 服务器验证通过，创建/登录用户

### 费用说明

短信验证费用取决于短信服务商：
- **阿里云短信**：约 ¥0.045/条
- **腾讯云短信**：约 ¥0.055/条
- **其他服务商**：价格不同

详见各短信服务商官网定价

## ⚠️ 注意事项

### 1. 生产环境必须禁用测试模式

```dart
// ❌ 错误：生产环境不能使用测试模式
static const bool enableTestPhoneNumber = true;

// ✅ 正确：生产环境必须禁用测试模式
static const bool enableTestPhoneNumber = false;
```

### 2. 短信发送限制

服务器端应配置以下限制：
- 同一手机号每天最多发送 5 次验证码
- 同一 IP 地址每小时最多发送 10 次验证码
- 超出限制会返回 `too-many-requests` 错误

### 3. 验证码有效期

验证码有效期为 **5 分钟**，过期后需要重新发送。

## 🐛 常见问题

### Q1: 真实手机号收不到验证码？

**可能原因**：
1. 服务器端短信服务未配置
2. 手机号格式不正确（必须包含国家代码，如 `+86`）
3. 短信被运营商拦截
4. 超出服务器发送限制

**解决方案**：
1. 检查服务器端短信服务配置
2. 确认手机号格式：`+[国家代码][手机号]`
3. 检查手机短信拦截设置
4. 等待一段时间后重试

### Q2: 提示 "too-many-requests" 错误？

**原因**：超出服务器发送限制

**解决方案**：
1. 等待 1 小时后重试
2. 使用不同的手机号测试
3. 切换到测试模式继续开发
