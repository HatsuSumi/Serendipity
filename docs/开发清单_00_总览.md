## 📝 开发规范

### 代码规范
1. **严格对照文档开发**
   - 开发前：列出所有要实现的字段/功能清单
   - 开发中：边写边对照文档
   - 开发后：逐项检查是否遗漏
   - 提交前：再次完整对照文档

2. **命名规范**
   - 类名：大驼峰（PascalCase）
   - 变量名：小驼峰（camelCase）
   - 常量名：全大写下划线（UPPER_SNAKE_CASE）
   - 文件名：小写下划线（snake_case）

3. **注释规范**
   - 类注释：说明类的用途
   - 方法注释：说明方法的功能、参数、返回值
   - 复杂逻辑：添加行内注释

4. **代码组织**
   - 一个文件一个类（除非是紧密相关的小类）
   - 按功能模块组织目录
   - 避免循环依赖

### Git 规范
1. **分支命名**
   - feature/功能名称
   - bugfix/问题描述
   - release/版本号

2. **提交信息**
   - feat: 新功能
   - fix: Bug修复
   - docs: 文档更新
   - style: 代码格式调整
   - refactor: 重构
   - test: 测试
   - chore: 构建/工具变动

3. **提交频率**
   - 完成一个小功能就提交
   - 避免一次提交过多改动
