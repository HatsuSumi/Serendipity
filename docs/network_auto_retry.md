# 网络恢复自动重试功能

## 📋 功能概述

当用户在离线状态下操作（如地铁、电梯），网络恢复后自动触发数据同步，确保数据安全。

## 🎯 设计原则

### 1. 单一职责原则（SRP）
- `NetworkMonitorService` 只负责监听网络状态
- 不直接操作 UI，不处理业务逻辑
- 通过 `SyncService` 触发同步

### 2. 依赖倒置原则（DIP）
- 通过 `Ref` 获取依赖，不直接依赖具体实现
- 使用 Provider 注入依赖

### 3. Fail Fast
- 参数验证立即抛出异常
- 避免重复监听

### 4. 分层架构
- Service 层：`NetworkMonitorService` 监听网络
- Service 层：`SyncService` 处理同步
- Provider 层：`authProvider` 提供用户状态

## 🏗️ 架构设计

```
NetworkMonitorService
  ↓ 监听网络状态变化
  ↓ 检测到网络恢复
  ↓ 检查用户是否登录 (authProvider)
  ↓ 触发同步 (SyncService)
  ↓ 上传本地数据 + 下载云端数据
```

## 🔧 实现细节

### 1. NetworkMonitorService

**职责：**
- 监听网络状态变化
- 检测网络从离线恢复到在线
- 检查服务器是否可达
- 触发自动同步

**关键方法：**
```dart
void startMonitoring(Ref ref)           // 开始监听
void stopMonitoring()                   // 停止监听
Future<bool> _checkServerHealth()       // 检查服务器健康状态
```

**设计亮点：**
- 使用 `Future.microtask` 避免在 Provider 构建期间触发同步
- 避免频繁触发（只在离线→在线时触发）
- **先检查服务器是否可达，再触发同步**（避免无效尝试）
- 同步失败不影响应用运行（静默失败）

**服务器健康检查：**
```dart
// 使用后端的 /health 端点
GET http://localhost:3000/health

// 成功响应：
{
  "message": "Server is running",
  "timestamp": "2026-03-08T...",
  "uptime": 12345.67
}

// 5 秒超时，失败返回 false
```

### 2. main.dart 集成

**修改：**
- `MyApp` 从 `ConsumerWidget` 改为 `ConsumerStatefulWidget`
- 在 `initState` 中启动网络监听
- 在 `dispose` 中停止网络监听

**设计亮点：**
- 使用 `addPostFrameCallback` 避免在构建期间访问 Provider
- 使用 `ref.onDispose` 自动清理资源


## 📊 性能优化

### 1. 避免频繁触发

**问题：** WiFi 和移动网络切换会触发多次网络状态变化

**解决：** 只在离线→在线时触发同步

```dart
if (isOnline && _wasOffline) {
  _onNetworkRestored(ref);
}
```

### 2. 避免无效同步

**问题：** 系统网络连接 ≠ 能访问服务器

**解决：** 先检查服务器健康状态

```dart
// 先 ping 服务器
final isServerHealthy = await _checkServerHealth();
if (!isServerHealthy) {
  return; // 服务器不可达，不触发同步
}
```

**优势：**
- ✅ 避免酒店 WiFi 场景的无效同步
- ✅ 避免服务器维护时的无效同步
- ✅ 减少网络流量
- ✅ 提升用户体验

### 3. 避免阻塞

**问题：** 同步可能耗时较长，阻塞 UI

**解决：** 使用 `Future.microtask` 异步执行

```dart
Future.microtask(() async {
  await _triggerSync(ref, user);
});
```

### 4. 静默失败

**问题：** 同步失败影响应用运行

**解决：** 捕获异常，不向上抛出

```dart
try {
  await syncService.syncAllData(user);
} catch (e) {
  // 静默失败，不影响应用运行
}
```

### 我们的实现（推荐）

```
connectivity_plus 监听网络
  ↓
检测到网络恢复
  ↓
Ping 自己的服务器 (/health)
  ↓
服务器可达 → 触发同步 ✅
服务器不可达 → 不触发同步 ❌
```

**优势：**
- ✅ 检测的是"能否访问自己的服务器"（最准确）
- ✅ 不依赖第三方服务（google.com 在中国被墙）
- ✅ 可以同时检测服务器状态
- ✅ 实现成本低（后端已有 /health 端点）
